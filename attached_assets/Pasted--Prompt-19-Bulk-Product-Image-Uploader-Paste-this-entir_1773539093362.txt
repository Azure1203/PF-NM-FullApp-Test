# Prompt 19 — Bulk Product Image Uploader

Paste this entire prompt into Replit.

We need the ability to upload a large batch of product images at once and have them automatically match to products by filename. This works for BOTH the `allmoxy_products` table (pricing/configurator products like shelves, panels, doors) and the `products` table (hardware products like handles, hinges, slides). The user drops a folder of images, the system shows which files matched which products, and on confirm it saves them all.

## Part 2 — Backend: Bulk Image Upload Endpoint

Add a new route to `server/routes.ts`:

```
POST /api/admin/products/bulk-upload-images
```

This route accepts a `multipart/form-data` request with multiple image files. Use the existing `multer` setup (already in the project). Configure multer to accept up to 200 files at once with a field name of `images`.

```ts
const bulkImageUpload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: 5 * 1024 * 1024, files: 200 }, // 5MB per file, 200 max
  fileFilter: (req, file, cb) => {
    const allowed = ['image/jpeg', 'image/png', 'image/webp', 'image/gif'];
    cb(null, allowed.includes(file.mimetype));
  }
});
```

The route logic:

1. Receive the uploaded files from `req.files`
2. Also receive a `target` field from `req.body` — either `"allmoxy"`, `"hardware"`, or `"both"` (default: `"both"`)
3. For each uploaded file:
   a. Extract the filename without extension: e.g. `34SHFF.jpg` → `34SHFF`, `H.801.13.201.png` → `H.801.13.201`, `801.13.201.png` → `801.13.201`
   b. Generate **search variants** from the extracted filename to handle supplier prefix conventions. Many product codes in the hardware table have an `H.`, `M-`, `M.`, `S.`, or `R.` prefix, but image filenames may or may not include that prefix. For each filename, build an array of variants to try:
      - The original filename as-is (e.g. `801.13.201`)
      - With `H.` prepended (e.g. `H.801.13.201`)
      - With `M-` prepended (e.g. `M-801.13.201`)
      - With `S.` prepended (e.g. `S.801.13.201`)
      - With `R.` prepended (e.g. `R.801.13.201`)
      - If the filename already starts with `H.`, `M-`, `M.`, `S.`, or `R.` — also try with that prefix stripped (e.g. `H.801.13.201` → also try `801.13.201`)
      
      This means a file named `801.13.201.png` will match hardware product `H.801.13.201`, and a file named `H.801.13.201.png` will also match. Both directions work.
      
   c. Try to match against products using this priority (try ALL variants at each step before moving to the next):
      - **Exact match on Allmoxy product name/skuPrefix** — if any variant (case-insensitive) exactly matches an `allmoxy_products.name` or `allmoxy_products.skuPrefix`, match it
      - **Exact match on Hardware product code** — if any variant exactly matches a `products.code`, match it
      - **Prefix match on Allmoxy** — if any variant starts with an Allmoxy product's `skuPrefix` (e.g. filename `34SHFF_front.jpg` matches product with skuPrefix `34SHFF`)
      - **Partial match** — if any variant contains a product name/code as a substring, or if a product name/code contains any variant as a substring
   d. Store the image using the Replit Object Storage service (already used in the codebase for packing slip images):
      ```ts
      const { ObjectStorageService } = await import('./replit_integrations/object_storage');
      const objectStorage = new ObjectStorageService();
      const storagePath = `product-images/${file.originalname}`;
      await objectStorage.uploadBuffer(storagePath, file.buffer);
      ```
   e. Record the match result: `{ filename, storagePath, matchedTable ('allmoxy' | 'hardware' | null), matchedProductId, matchedProductName, confidence ('exact' | 'prefix' | 'partial' | 'none') }`

4. **Do NOT save the imagePath to any product yet.** Return all match results as a preview:

```json
{
  "results": [
    {
      "filename": "34SHFF.jpg",
      "storagePath": "product-images/34SHFF.jpg",
      "matchedTable": "allmoxy",
      "matchedProductId": 42,
      "matchedProductName": "34SHFF",
      "confidence": "exact"
    },
    {
      "filename": "H.801.13.201.png",
      "storagePath": "product-images/H.801.13.201.png",
      "matchedTable": "hardware",
      "matchedProductId": 367,
      "matchedProductName": "H.801.13.201",
      "confidence": "exact"
    },
    {
      "filename": "801.13.201.png",
      "storagePath": "product-images/801.13.201.png",
      "matchedTable": "hardware",
      "matchedProductId": 367,
      "matchedProductName": "H.801.13.201",
      "confidence": "exact"
    },
    {
      "filename": "random_photo.jpg",
      "storagePath": "product-images/random_photo.jpg",
      "matchedTable": null,
      "matchedProductId": null,
      "matchedProductName": null,
      "confidence": "none"
    }
  ],
  "summary": {

  }
```

## Part 4 — Backend: Serve Product Images

Add a route to serve images from object storage:

```
GET /api/product-images/:imagePath(*)
```

This route reads the image from Replit Object Storage and streams it back with the correct Content-Type header. Add cache headers (`Cache-Control: public, max-age=86400`).

```ts
app.get('/api/product-images/*', isAuthenticated, async (req, res) => {
  try {
    const imagePath = req.params[0];
    // Validate path to prevent traversal
    if (imagePath.includes('..') || imagePath.startsWith('/')) {
      return res.status(400).json({ message: 'Invalid path' });
    }
    const fullPath = `product-images/${imagePath}`;
    const { ObjectStorageService } = await import('./replit_integrations/object_storage');
    const objectStorage = new ObjectStorageService();
    const buffer = await objectStorage.downloadBuffer(fullPath);
    if (!buffer) {
      return res.status(404).json({ message: 'Image not found' });
    }
    // Detect content type from extension
    const ext = imagePath.split('.').pop()?.toLowerCase();
    const contentTypes: Record<string, string> = {
      jpg: 'image/jpeg', jpeg: 'image/jpeg', png: 'image/png',
      webp: 'image/webp', gif: 'image/gif'
    };
    res.setHeader('Content-Type', contentTypes[ext || ''] || 'image/jpeg');
    res.setHeader('Cache-Control', 'public, max-age=86400');
    res.send(buffer);
  } catch (err: any) {
    res.status(500).json({ message: 'Failed to serve image' });
  }
});
```
