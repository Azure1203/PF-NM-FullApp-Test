We are building a new order management application. The current codebase already has a working database schema in shared/schema.ts with tables for allmoxy_products, attribute_grids, attribute_grid_rows, proxy_variables, color_grid, projects, order_files, and others.
We need to make three schema additions. Do not change or remove anything that already exists. Only add the following:
1. Add three new columns to the existing allmoxy_products table:

sku_prefix — varchar(100), nullable. This is the SKU prefix string used to match CSV line items to this product (e.g. "34SHFF", "TK", "FILL"). A line item whose SKU starts with this prefix belongs to this product.
description — text, nullable. A plain-language description of what this product is, for documentation purposes.
notes — text, nullable. Internal notes about the product's pricing logic or special handling.

2. Create a new product_grid_bindings table:
This table links a product to the attribute grids it uses when evaluating its pricing formula. Each binding has:

id — serial primary key
product_id — integer, foreign key to allmoxy_products(id), ON DELETE CASCADE, not null
grid_id — integer, foreign key to attribute_grids(id), ON DELETE CASCADE, not null
alias — varchar(100), not null. The variable name used in the formula scope for this grid's data (e.g. "color", "mjdoors", "shelves"). When the pricing engine runs, the matched row from this grid will be available in the formula as alias.COLUMN_NAME.
lookup_column — varchar(100), not null. The column name in the CSV order row that provides the lookup key for this grid (e.g. "Color", "MANU_CODE", "SKU").

3. Create a new order_items table:
This stores the per-line-item pricing results after an order CSV is processed. Each row represents one line item from the CSV:

id — serial primary key
project_id — integer, foreign key to projects(id), ON DELETE CASCADE, not null
file_id — integer, foreign key to order_files(id), ON DELETE CASCADE, not null
product_id — integer, foreign key to allmoxy_products(id), ON DELETE SET NULL, nullable. Null if no product matched.
sku — text, nullable. The SKU/MANU_CODE from the CSV row.
description — text, nullable. The item name/description from the CSV row.
width — real, nullable.
height — real, nullable.
depth — real, nullable.
quantity — integer, default 1.
unit_price — real, default 0.
total_price — real, default 0.
export_text — text, nullable. The generated Cabinet Vision .ORD block for this item.
pricing_error — text, nullable. If pricing failed, store the error message here so it can be displayed in the UI.
raw_row_data — jsonb, nullable. The full original CSV row stored as JSON for reference and re-processing.
created_at — timestamp, default now().
