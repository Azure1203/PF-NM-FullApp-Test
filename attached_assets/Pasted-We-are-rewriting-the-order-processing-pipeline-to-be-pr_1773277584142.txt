Background: How the CSV is structured
The customer order CSV has a header section at the top (dealer info, shipping address, PO number, etc.), followed by a data section. The data section starts on the row after the row whose first column contains the word "Manuf" (case-insensitive). Each data row has this column layout:

Column 0 (row[0]): SKU / MANU_CODE — the part code, e.g. 34SHFF, TK, DBX..., H.801.43...
Column 1 (row[1]): Color code — e.g. 300, K79, MT52
Column 2 (row[2]): Quantity
Column 3 (row[3]): Height (in mm)
Column 4 (row[4]): Width (in mm)
Column 5 (row[5]): Thickness / Depth (in mm)

When the CSV is parsed with columns: true, the named columns used by the pricing engine are whatever the header row says — typically MANU_CODE, Color, Qty, Height, Width, Thickness.

Step 2 — Write a matchProductToSku helper function
Add this function inside routes.ts (above the route handler, not exported):
typescriptfunction matchProductToSku(sku: string, activeProducts: AllmoxyProduct[]): AllmoxyProduct | null {
  if (!sku) return null;
  const upperSku = sku.toUpperCase().trim();
  // Find the product whose skuPrefix is the longest match at the start of the SKU
  // This handles cases where one prefix is a substring of another (e.g. "34SHF" vs "34SHFF")
  let bestMatch: AllmoxyProduct | null = null;
  let bestLength = 0;
  for (const product of activeProducts) {
    const prefix = (product.skuPrefix || '').toUpperCase().trim();
    if (prefix && upperSku.startsWith(prefix) && prefix.length > bestLength) {
      bestMatch = product;
      bestLength = prefix.length;
    }
  }
  return bestMatch;
}
