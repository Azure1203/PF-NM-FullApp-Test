FIX 1 — Use product.skuPrefix as fallback for MANU_CODE bindings
(server/routes.ts — formula-test endpoint)
═══════════════════════════════════════════════════

The lookupValue resolution for grid bindings currently falls back to 
product.name when no explicit value is provided. product.name is a 
human-readable label (e.g. "15\" Divider Panel (TFL)") that does not 
match the MANU_CODE in the grid.

For bindings where lookupColumn contains "manu" (case-insensitive), 
the correct auto-resolve value is product.skuPrefix — which IS the 
MANU_CODE (e.g. "15DIVFBC").

Change the lookupValue resolution from:
  const lookupValue = String(
    (req.body.inputs || {})[binding.lookupColumn] ||
    (req.body.gridLookups || {})[binding.alias] ||
    (req.body.gridLookups || {})[binding.lookupColumn] ||
    product.name ||
    ''
  );

To:
  const isManuCodeBinding = binding.lookupColumn
    .toLowerCase().includes('manu');
  const autoValue = isManuCodeBinding 
    ? (product.skuPrefix || product.name || '')
    : (product.name || '');
  const lookupValue = String(
    (req.body.inputs || {})[binding.lookupColumn] ||
    (req.body.gridLookups || {})[binding.alias] ||
    (req.body.gridLookups || {})[binding.lookupColumn] ||
    autoValue
  );

Also fix: when a lookup returns no row, do NOT set 
contextScope[binding.alias] = null. Simply don't set it at all:

  if (row) {
    const rawData = row.rowData as Record<string, any>;
    contextScope[binding.alias.toLowerCase()] = Object.fromEntries(
      Object.entries(rawData).map(([k, v]) => [k.toLowerCase(), v])
    );
  }
  // No else — leaving alias out of scope gives a clearer error
  // than setting it to null which causes "Cannot read properties of null"
